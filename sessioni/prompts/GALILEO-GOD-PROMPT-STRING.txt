## WORKFLOW — RALPH LOOP

Esegui cicli iterativi:

### Per ogni ciclo:

**STEP 1 — IMPLEMENTA** una fase dal piano (nell'ordine: 1,2,3,4,6,7). Leggi il piano completo in .claude/plans/agile-floating-whistle.md per i dettagli tecnici di ogni fase.

**STEP 2 — BUILD + DEPLOY**
- Frontend: cd "VOLUME 3/PRODOTTO/elab-builder" && npm run build && npx vercel --prod --yes
- Backend (se modifiche server.py/nanobot.yml): cd nanobot && git add -A && git commit -m "fix: ..." && git push
- ZERO errori build richiesto

**STEP 3 — TEST (Chrome Control)**
Apri https://www.elabtutor.school in Chrome. Naviga a /#tutor, apri chat Galileo.
Esegui un batch di 30 domande dal campione (vedi file sessioni/prompts/GALILEO-GOD-MEGA-PROMPT.md per le 300 domande complete divise in 5 personas x 60 domande).

Le 5 personas:
1. Marco (9 anni, principiante assoluto, scrive con errori)
2. Sofia (12 anni, intermedia, ha fatto Vol1)
3. Prof. Rossi (insegnante non tecnico, usa ELAB in classe)
4. Alex (13 anni, scherzoso, testa limiti, fa battute)
5. Luca (16 anni, esperto tecnico, vuole cose avanzate)

Per ogni domanda: digita -> attendi risposta -> valuta PASS/FAIL/PARTIAL -> documenta.

**STEP 4 — ANALISI**
Calcola success rate, categorizza FAIL, identifica pattern.

**STEP 5 — FIX pattern errore**
- Problema nel PROMPT -> modifica nanobot.yml
- Problema nel ROUTING -> modifica server.py
- Problema nel FRONTEND parser -> modifica ElabTutorV4.jsx
- Problema nell'UI -> modifica ChatOverlay.jsx / NewElabSimulator.jsx
- Problema nella PHYSICS -> modifica CircuitSolver.js

**STEP 6 — RE-TEST** domande fallite, poi prossimo batch.

Dopo ogni 2 cicli (60 domande), analisi con team agenti:
- Agente UI/UX, Agente AI, Agente Web, Agente ELAB, Agente QA
- Ciascuno: 3 punti forti, 3 problemi critici, 3 suggerimenti, score 1-10
