## Design Context

### Users

**Primario — Docente scuola primaria/secondaria 1° grado (8-14 età studenti)**:
- Insegnante non-tecnico tipico, NON sviluppatore. Spesso prima-volta con Arduino + elettronica.
- Contesto uso: aula scuola pubblica/paritaria + LIM (Lavagna Interattiva Multimediale) proiettata 65"-86" front-class.
- Non-conoscenze pregresse: deve capire SUBITO senza training (CHIUNQUE accenda ELAB Tutor deve giostrarsi senza help). NO onboarding 30 min.
- Job-to-be-done: preparare lezione + spiegare ai ragazzi + dimostrare circuito + verificare comprensione + assegnare esercizi kit-fisici.
- Stack mediato: docente legge volume cartaceo + proietta UNLIM su LIM + ragazzi montano kit Omaric breadboard reale + rispondono.

**Secondario — Studenti 8-14 (vedono via LIM, NON interagiscono direttamente UNLIM)**:
- Bambini scuola primaria 4ª-5ª + secondaria 1° grado 1ª-3ª.
- Manuali + smartphone-first generation MA NON gaming-addicted.
- Vedono LIM proiettata, lavorano kit fisico breadboard + componenti reali.
- NON usano UNLIM direttamente: il docente è tramite (PRINCIPIO ZERO).

**Terziario — Tea/collaboratrice + Andrea (developer)**:
- Power user manutenzione contenuti + struttura volumi.
- Workspace dashboard docente + sync Supabase classi.

### Brand Personality

**3 parole**: **Affidabile / Didattico / Accogliente** (Andrea-confirmed iter 8 close 2026-04-27).

**Voce + tono**:
- Affidabile = il software FUNZIONA, NON crasha LIM in classe. Errori chiari, fallback graceful, recovery sempre possibile. Promise zero demo / dati finti (CLAUDE.md regola immutabile §12).
- Didattico = lessico Italiano scuola pubblica, plurale "Ragazzi," (PRINCIPIO ZERO), citazione Vol/pag VERBATIM volumi cartacei (NO parafrasi). Analogie real-world ("LED come lampadina", "resistore come tappo bottiglia"). MAX 60 parole risposta. NO imperativo docente.
- Accogliente = onboarding zero-frizione. Toast errori in italiano caldi NON tecnici. Mascotte UNLIM (`logo-senza-sfondo.png`) come compagno NON robot freddo. Spazio per errore + retry, mai punizione.

**Goal emozionale**: docente sente "ce la posso fare anche se non sono ingegnere" + studente sente "voglio costruire questo".

### Aesthetic Direction

**Direzione visuale dominante** (inferita codebase + Morfismo dottrina):

**Stile**: kit-coupled didactic. Software è MORFICO al kit fisico Omaric + 3 volumi cartacei (CLAUDE.md "DUE PAROLE D'ORDINE §2 MORFISMO"). NON è "qualsiasi educational SaaS" — è "il software del kit + volumi". Coerenza estetica triplet → moat 2026+ vs LLM-generated copycat.

**Reference positivi inferiti**:
- **Lego Education / Geomag**: kit-coupled physical-first, componenti tangibili, palette bold ma controllata, step visuali 1:1 con materiale fisico (Morfismo DUAL SENSE Sense 2 — triplet coerenza).
- **Khan Academy soft layout**: whitespace generoso + tipografia umana (Open Sans body + Oswald heading) + cards arrotondate.
- **Volumi cartacei ELAB stessi**: stessa palette (Navy/Lime/Orange/Red = palette stampa volumi) + stesse iconografie + stessi nomi capitoli.

**Anti-reference espliciti** (CLAUDE.md "Anti-pattern Morfismo vietati"):
- ❌ Generico Material Design SaaS (Google Material standard FAB blu piatti, qualsiasi-prodotto). Non passa Test Morfismo "stesso prodotto del Volume?".
- ❌ Cartoon kiddie / Disney / asilo (arcobaleni eccessivi, comic-sans, mascotte caricaturali). Bambini 8-14 NON sono primaria 1ª.
- ❌ Enterprise B2B Salesforce (dashboard grigie corporate, widget metric overload, palette aziendale fredda). Scuola NON è ufficio.
- ❌ Gaming RGB hyper-stimolante (particelle, timer urgency, loot box, dopamine hit). Educational NON è arcade.
- ❌ Componenti SVG palette generica (NO blu/rosso standard). DEVE essere palette stampa volumi.
- ❌ UNLIM parafrasa volumi (DEVE citare VERBATIM Vol.X pag.Y).
- ❌ Capitoli software titoli inventati (DEVE riusare titoli libro fisico — NO "Lesson 1", SI "Capitolo 6 — I LED").
- ❌ Esercizi software-only (DEVE 1:1 mapping libro fisico).
- ❌ Icone material-design generiche (DEVE usare `ElabIcons.jsx` derivate volumi, MAI emoji).
- ❌ Layout simulator non riconducibile al kit fisico.

**Theme**:
- **Light mode primario obbligatorio** (LIM proiezione contesto dominante — dark mode peggiora contrasto proiettore).
- Dark mode opzionale auto-sistema future (P3, non blocca iter 9-10).

**Palette tokens** (`src/styles/design-system.css`):
- `--color-primary: #1E4D8C` Navy (UNLIM brand + capitoli + headings)
- `--color-accent: #4A7A25` Lime (Vol 1 + accent positivi + success)
- `--color-vol1: #4A7A25` Lime (Volume 1 — Elettronica base)
- `--color-vol2: #E8941C` Orange (Volume 2 — Componenti) + `--color-vol2-text: #996600` (WCAG AA 4.94:1 text-on-white)
- `--color-vol3: #E54B3D` Red (Volume 3 — Arduino) + `--color-vol3-text: #C62828` (WCAG AA 7.1:1 text-on-white)
- `--color-danger: #DC2626` Red error
- `--color-warning: #EA580C` Orange warn
- `--color-success: #16A34A` Green confirm
- `--color-bg: #FFFFFF` (light only)

**Tipografia tokens**:
- `--font-heading: 'Oswald', -apple-system, sans-serif` — titoli, capitoli, mascotte. Compresso + autoritativo.
- `--font-sans: 'Open Sans', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` — body, leggibilità lunga, accessibile.
- `--font-display: 'Oswald', 'Arial Narrow', Arial, sans-serif` — display large numbers/volumi.
- `--font-mono: 'Fira Code', 'SF Mono', 'Consolas', monospace` — codice Arduino C++.
- Variable fonts loaded `src/styles/fonts.css` (Open Sans 300-800 + italic).

**Tipografia regole** (CLAUDE.md "Qualita' §8-§10"):
- Min 13px testi corpo + 10px label secondarie.
- Touch target minimo 44×44px bottoni interattivi.
- Contrasto WCAG AA: 4.5:1 testo, 3:1 grafici.

### Design Principles (5 pilastri ELAB Tutor)

1. **Morfismo Triplet Coerenza + Adattabilità docente/classe/funzioni** (Sense 1+2+1.5 iter 10):
   - Software ↔ kit Omaric ↔ volumi cartacei = singola esperienza (Sense 2 triplet)
   - Adatta runtime per-docente (linguaggio INVARIATO scuola pubblica + plurale Ragazzi, MA dettagli + memoria docente specifici)
   - Adatta contesto classe (età studenti + livello + kit dotazione + capitolo corrente + LIM/iPad config)
   - Funzioni morfiche (stessa capacity, presentazione adattiva: Lavagna lezione vs Dashboard analisi)
   - Finestre morfiche (risoluzione LIM + mode active + touch/voice + stato sessione)
   - Toolbar + mascotte + quick-access pannelli adattivi
   - Test pre-merge: pagina random Volume + schermata random software → "stesso prodotto"? Se NO → REJECT. + Test docente A vs docente B stesso esperimento stessa LIM → layout/funzioni adattate identità+storia.
   - (CLAUDE.md DUE PAROLE D'ORDINE §2 Sense 1+1.5+2 + sprint-S-iter-8-contract.md DUAL SENSE)

2. **Principio Zero Pedagogico** — docente è tramite, UNLIM è strumento, kit fisico è protagonista. Plurale "Ragazzi," + cita Vol/pag VERBATIM + ≤60 parole + analogia real-world + mai imperativo docente. Test runtime: ogni risposta UNLIM passa `_shared/principio-zero-validator.ts` 6 rules. (CLAUDE.md DUE PAROLE D'ORDINE §1).

3. **LIM-First Light Mode** — ottimizza proiezione 65-86" front-class luminosa. Light mode primario obbligatorio. Touch target 44×44px (proiezione interattiva touch). Tipografia leggibile a 5m distanza. NO dark mode default.

4. **Mai Demo Mai Mock** — zero dati finti, zero placeholder, zero "funzionerà più tardi". Tutto live con dati reali Supabase + Edge Functions + RAG 1881 chunks + Wiki 100/100. Errori chiari italiani caldi non-tecnici. Fallback graceful chain (callLLMWithFallback Together → Gemini → Together gated). (CLAUDE.md regola §12).

5. **Anti-Inflation CoV** — claim numeri verificati live (`vitest run` output OR `automa/baseline-tests.txt`), MAI a memoria. "Quasi pronto" = 0%. "Funziona" = testato live utente reale. Score oggettivo `node scripts/benchmark.cjs` solo verità. CoV su ogni PR: 3× vitest run prima dichiarare PASS. (CLAUDE.md "Regola anti-inflazione numerica").

## Anti-pattern checklist pre-merge (test Morfismo)

Ogni feature iter 9-10+ verifica:

### Sense 2 — Triplet coerenza materiale

| Check | PASS criteria |
|-------|---------------|
| Palette generica | NO Material blu/rosso standard. SI tokens `--color-primary`/`--color-vol1-3` |
| UNLIM parafrasa | NO. SI cita `Vol.X pag.Y "testo esatto"` |
| Capitoli inventati | NO "Lesson 1". SI "Capitolo 6 — I LED" (titolo libro) |
| Esercizi software-only | NO. SI 1:1 mapping volumi (lesson-paths/v1-cap6-esp1) |
| Emoji icone | NO. SI `ElabIcons.jsx` |
| Layout non-kit | NO. SI NanoR4Board SVG = Arduino kit Omaric pixel-perfect |
| Touch target | ≥44×44px |
| Font min | ≥13px body, ≥10px label |
| Contrasto | WCAG AA 4.5:1 testo, 3:1 grafici |
| LIM-first | Light mode primario verificato 65-86" projection |

### Sense 1+1.5 — Morfismo runtime docente/classe/funzioni (NEW iter 10)

| Check | PASS criteria |
|-------|---------------|
| Linguaggio invariato | Plurale "Ragazzi," + cita Vol/pag VERBATIM **NON cambia** per-docente. Cambia DETTAGLI + memoria + esempi. |
| Adattabilità docente | Memoria docente persiste sessioni (Supabase) + suggerimenti next-step coerenti percorso fatto + stile rilevato |
| Adattabilità classe | Età studenti + livello + kit dotazione + capitolo corrente + LIM/iPad config influenzano output |
| Funzioni morfiche | Stesse capacità (highlight, mountExperiment, captureScreenshot) MA presentazione adatta mode (Lezione/Lavagna/Dashboard) |
| Finestre morfiche | Pannelli/overlay/floating adattano risoluzione LIM + mode + touch/voice + stato sessione |
| Toolbar morfica | 4 core tools layout/icone scalano LIM front-class vs iPad student |
| Mascotte UNLIM | Posizione + dimensione + stato (parla/ascolta/aspetta) adatta workflow |
| Quick-access pannelli | Shortcut docente frequenti rilevati automaticamente runtime |
| Layout adattivo | Stesso schermo LIM proietta DIVERSO per docente A vs B (preferenze rilevate, NON config manual) |
| Static-config | NO presets fissi tutti utenti. SI runtime self-adapting ogni docente+classe |
| Test docenti diversi | Stesso esperimento aperto da 2 docenti diversi su stessa LIM → layout/funzioni/finestre adattate identità+storia ciascuno |

Se feature viola UN solo check (Sense 1+1.5+2) → REJECT pre-merge.
