1. Sommario esecutivo

ELAB Tutor (elabtutor.school) è un tutor educativo per elettronica e Arduino destinato a bambini di 8–14 anni nelle scuole italiane. Il prodotto unifica software web + kit fisici Omaric + tre volumi cartacei (autore Davide Fagherazzi). La componente AI UNLIM richiede un mix di servizi: LLM, VLM, TTS, STT, embeddings, image generation.

Stack scelto iter 26 (decisione)

  • LLM primario: Mistral routing 65/25/10 (Mistral Small 3.1 / Mistral Large / Llama 3.3 70B Together AI fallback)
  • Vision (VLM): Mistral Pixtral 12B / Pixtral Large
  • STT: Cloudflare Whisper Turbo
  • TTS: edge-tts Isabella (baseline gray-area), Coqui XTTS-v2 self-host (RunPod sandbox iter 28)
  • Image gen: Cloudflare FLUX.1 Schnell
  • Embeddings: mistral-embed primario, BGE-M3 via CF fallback
  • Sandbox dev/bench: RunPod RTX A6000 / RTX 6000 Ada (on-demand, frugale)

2. Glossario — termini e aziende

2.1 Termini tecnici

LLM (Large Language Model)
Modello linguistico di grandi dimensioni. Esempi: Mistral Large, Llama 3.3 70B, Gemini Pro, GPT-4o. Misurato in parametri (es. 70B = 70 miliardi).
VLM (Vision-Language Model)
Modello che capisce immagini + testo. Esempi: Pixtral 12B, Qwen2.5-VL, GPT-4o.
TTS (Text-to-Speech)
Sintesi vocale: testo → audio. Esempi: ElevenLabs, Cartesia, Coqui XTTS, edge-tts.
STT (Speech-to-Text)
Riconoscimento vocale: audio → testo. Esempi: Whisper, Whisper Turbo, Deepgram.
RAG (Retrieval Augmented Generation)
Pattern dove l'LLM riceve documenti rilevanti prima di rispondere. Riduce allucinazioni.
Embedding
Vettore denso che rappresenta semanticamente un testo. BGE-M3 produce vettori dimensione 1024.
BGE-M3
BAAI General Embedding M3: modello embedding multilingue 567M parametri, MIT license, 100+ lingue (italiano nativo).
RRF (Reciprocal Rank Fusion)
Algoritmo di fusione di ranking. Formula: rrf(d) = Σᵢ 1/(k + rankᵢ(d)) con k=60 (Cormack et al., SIGIR 2009).
Token
Unità di testo (~4 caratteri italiani). Pricing LLM è quasi sempre per milione di token.
Pricing input/output
LLM addebitano separatamente token prompt (input) e token generati (output). Output costa di norma 2–4× input.
Self-host
Eseguire il modello su infrastruttura propria (es. GPU su RunPod). Contrario di API.
GGUF
Formato quantizzazione modelli (es. Q4_K_M comprime un 7B da 14GB a 4GB).
On-demand vs Spot
On-demand = prezzo fisso, sempre disponibile. Spot/Community = ~50% più economico, può essere interrotto.
GDPR / DPA
General Data Protection Regulation EU. DPA = Data Processing Agreement (contratto firmato col vendor per trattare dati personali, obbligatorio per minori).

Flusso query: 7-layer → RRF k=60 → Mistral → Risposta

3.3 Note di implementazione

  • Layer 1+2+7 recuperati tramite ricerca ibrida: BM25 (sparse, keyword) + dense embedding (BGE-M3 dim 1024) fusi via RRF k=60. Riferimento: supabase/functions/_shared/rag.ts:958 LOC (iter 12 OR-fallback 2-token threshold).
  • Layer 3 usa la chiave classe (class_key in localStorage) per recuperare ultime 10 sessioni della stessa classe.
  • Layer 4 contesto vivo (lesson-paths-narrative + state simulatore + ultima azione).
  • Layer 5+6 iniettati direttamente nel system prompt come segmenti pre-formattati.
  • Context complessivo limitato a 128k token (Mistral Large) con fallback 32k (Mistral Small).
Caveat onesto: la fusione RRF è documentata e testata; il punteggio retrieval italiano K-12 non è ancora benchmarkato con gold-set proprio (iter 22 ha shipped fixture 30 query, esecuzione end-to-end deferita iter 27).

5. Casi possibili — 5 scenari

5.1 Caso A — MAX EU sovereign

Stack: Mistral Small 3.1 (LLM) + OVHcloud Llama 3.3 (fallback) + Coqui XTTS-v2 self-host + Aleph Alpha embedding (DE) + Scaleway H100 sandbox.

Pro: Massima sovranità EU, GDPR-pulito al 100%, narrativa marketing forte ("AI made in EU per scuole italiane").

Contro: Aleph Alpha Luminous $30/M = 30× più caro di mistral-embed. Costo a scala è proibitivo.

Costo stimato 1.000 classi/mese: ~€500 (~5× caso B).

5.2 Caso B — Hybrid EU+US DPA (vincente per ELAB iter 26)

Stack: Mistral routing 65/25/10 + Cloudflare (FLUX/Whisper/BGE-M3) + Together AI Llama fallback + Coqui RunPod sandbox.

Pro:

  • Mistral è EU sovereign (Parigi) per il primario
  • Cloudflare ha region EU + free tier generoso
  • Together AI fallback con DPA (US) usato solo gated (anonimizzato + log audit)
  • Costo competitivo

Contro: Together AI è US (DPA mitiga ma non elimina trasferimento). edge-tts gray-area TOS.

Costo stimato 1.000 classi/mese: ~€170 (vedi tabella sommario).

5.3 Caso C — Cheapest CF-heavy

Stack: Cloudflare Workers AI per tutto (LLM Llama via Workers AI + FLUX + Whisper + BGE-M3).

Pro: Singolo vendor, free tier 10k Neurons/giorno, region EU.

Contro: Cloudflare LLM ancora limitato (no Mistral, no Pixtral nativo); qualità italiano non benchmarkata K-12.

Costo stimato 1.000 classi/mese: ~€80–120 (free tier coperto fino a ~500 classi).

5.4 Caso D — All Scaleway managed

Stack: Scaleway Generative APIs (€0,20/M) + Managed Inference + GPU H100 dedicata.

Pro: Cloud sovrano FR completo, una sola fattura, datacenter Parigi. Predicibilità costi.

Contro: Catalogo modelli più stretto di Mistral diretto. Managed Inference richiede commit minimo.

Costo stimato 1.000 classi/mese: ~€200.

5.5 Caso E — Self-host VPS H100 (>10k classi)

Stack: VPS dedicato H100 (Scaleway o RunPod reserved) con vLLM Llama 3.1 70B + Coqui + BGE-M3 + Whisper.

Costo fisso: H100 ~€2,73/h × 24 × 30 = €1.965/mese fissi.

Break-even: ~10.000 classi attive (a costo variabile €0,17/classe equivalente API).

Per ELAB iter 26 (200–2.000 classi target): NON SCEGLIERE. Eccessivo.

9. Multi scenari — decisioni alternative + risk

9.1 Cosa succede se Mistral free tier rate-limit hits a scala 5k+?

Mistral PAYG (Pay-As-You-Go) non ha rate limit duro a livello account. Il rischio reale è che il tier gratuito sviluppo (3 RPM) sia insufficiente per benchmark batch.

Mitigazione:

  • Andrea ha already-ratified Mistral PAYG signup nel queue iter 22 (voce 4 ratify queue).
  • Backup: Together AI Llama 3.3 70B Turbo (provato iter 21 con 4/4 PZ V3 PASS) come fallback automatico via callLLMWithFallback.

9.2 Cosa succede se Cloudflare FLUX free tier 10k/giorno superato?

Cloudflare Workers AI: 10.000 Neurons/giorno (300k/mese) free, poi pay-per-Neuron.

A 5.000 classi attive: 5.000 immagini/giorno fattibile dentro free tier. A 10.000+ classi: pagamento ~€20–40/mese stimato.

Mitigazione: auto-disabilita FLUX per classi inattive (already implemented via lesson-paths active flag). Cache aggressiva immagini generate (già supportato Cloudflare R2).

9.3 Cosa succede se Davide Fagherazzi ratify slip Vol3 V0.9?

Iter 22-30 PDR §3.3 ha già documentato: Davide schedule slip = LOW probability, LOW impact (Vol3 V2 ground truth corretto a 0.92, no longer blocker).

Mitigazione: lesson-paths-narrative refactor è indipendente dal contenuto Davide; può essere lanciato senza Vol3 V0.9.

9.4 Cosa succede se RunPod budget esaurito sandbox dev?

Budget RunPod attuale: ~$11 residui (post iter 5 P3 pod TERMINATED + storage).

Mitigazione:

  • Discipline frugale: pod STOPPED immediato post-task, TERMINATE se non si riusa entro 2 settimane.
  • Coqui XTTS-v2 voice clone iter 28 stimato 4h × $0,49/h = $2 (dentro budget).
  • Se budget esce: Cartesia paid $0,03/min è il backup commerciale (~€15/mese a scala 5.000 classi).
