## 2. Issues encountered + solutions

### 2.1 NordVPN intercepts Tailscale (HIGH severity, blocked all progress)

**Symptom**: Tailscale admin shows Mac Mini "Connected" but ports 5900/22 timeout from MacBook.
**Root cause**: NordVPN active on MacBook intercepted Tailscale UDP traffic + DERP relay routes shifted to Singapore (Italian ISP routing through VPN).
**Solution**: Pause NordVPN. Long-term: NordVPN split tunneling exclude `100.64.0.0/10` Tailscale subnet.
**Lesson**: VPN + Tailscale conflict is silent — `tailscale netcheck` Nearest DERP is the diagnostic.

### 2.2 Wrong Tailscale account (MEDIUM)

**Symptom**: Browser tailscale admin showed 0 devices.
**Root cause**: Andrea logged into `marro.andrea96@gmail.com` admin, but devices belong to `progettidigetto@gmail.com` tailnet.
**Solution**: Switch admin login.
**Lesson**: Multi-account Tailscale = check `managedby` field carefully.

### 2.3 VNC keyboard typing unreliable (MEDIUM)

**Symptom**: typing `claude --version` via Screen Sharing produced `aaaaaaaaaa...` (char repeat).
**Root cause**: VNC keystroke delivery latency/buffering causes macOS key repeat to fire.
**Solution**: Use SSH passwordless from MacBook for all command typing. Reserve VNC for visual verification only.
**Lesson**: VNC = visual control. SSH = command control. Don't mix.

### 2.4 SSH key paste corruption (MEDIUM)

**Symptom**: SSH passwordless test failed even after key add.
**Root cause**: Earlier paste of public key included embedded newline (token wrap visual interpreted as actual newline).
**Solution**: Re-add via clean echo command (no copy-paste from VNC).
**Lesson**: For long strings, use clipboard fast-path or single-line echo, NEVER paste multi-line via VNC.

### 2.5 Local user vs email account confusion (LOW)

**Symptom**: SSH `progettidigetto@100.124.198.59` failed Permission Denied.
**Root cause**: Mac Mini local user is `progettibelli`, NOT `progettidigetto` (email of Apple ID/Tailscale).
**Solution**: Use `progettibelli@` for SSH.
**Lesson**: macOS local username ≠ Apple ID email. Verify via `whoami` first.

### 2.6 Keychain not accessible in SSH non-interactive (HIGH)

**Symptom**: `security find-generic-password -w` outputs empty file when run via SSH non-interactive.
**Root cause**: macOS Keychain GUI prompt doesn't render in SSH session → silent denial.
**Solution**: Run `security` command on Mac Mini interactive terminal first → GUI prompt allows access → token saved to file → SSH reads file (bypasses Keychain).
**Lesson**: Keychain = interactive only. For autonomous workflows, use file-based credential with restrictive perms.

### 2.7 OAuth token paste with embedded newline (HIGH)

**Symptom**: 401 Invalid bearer token after Keychain add.
**Root cause**: Token displayed across 2 visual lines, paste copied newline character. Keychain stored token with `\n` in middle.
**Solution**: Delete Keychain entry, re-run `claude setup-token`, paste WITHOUT newline (single line).
**Lesson**: Long tokens visually wrap — always single-line paste. Verify token length matches expected (~108 chars for Claude OAuth).

### 2.8 .zprofile vs .zshenv for SSH inheritance (LOW)

**Symptom**: Token env var set in `.zprofile` not loaded in SSH non-interactive session.
**Root cause**: `.zprofile` loaded by login shells only. Non-interactive SSH uses `.zshenv`.
**Solution**: Move env exports to `.zshenv` for SSH inheritance.
**Lesson**: `.zshenv` = always loaded. `.zprofile` = login only. `.zshrc` = interactive only.

### 2.9 GitHub credentials not on Mac Mini (RESOLVED by design)

**Symptom**: `git push` from Mac Mini fails GitHub authentication.
**Root cause**: Mac Mini has no GitHub PAT or SSH key for github.com.
**Solution**: SSH keys for GitHub stay on MacBook (Andrea decision). Workflow: Mac Mini commits locally → MacBook scp file → MacBook commits + pushes.
**Trade-off**: Adds 1 manual step but keeps all GitHub credentials single-source on MacBook.
