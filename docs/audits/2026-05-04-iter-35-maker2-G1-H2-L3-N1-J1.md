## Architectural decisions documented

### G1+G2+G3 — Libero TRUE clear via event-based decoupling

**Problem**: `clearAll()` cleared simulator runtime, but ComponentDrawer's `mode==='guided'` PRONTI banner still rendered because it derived state from `currentExperiment.buildMode` (read from NewElabSimulator, NOT LavagnaShell).

**Solution**:
1. LavagnaShell's `handleModalitaChange('libero')` dispatches `CustomEvent('elab-lavagna-libero-enter', { detail: { timestamp } })` so consumers can react without tight coupling.
2. ComponentDrawer reads `localStorage.getItem('elab-lavagna-libero-active') === 'true'` as **single source of truth** + listens to the CustomEvent for instant flip.
3. When `liberoActive && mode === 'guided'` → return null (defense-in-depth gate; suppresses guided UI including PRONTI banner).
4. Polling fallback (1s interval) catches edge cases where `storage` event doesn't fire in same tab.

**Trade-off**: ComponentDrawer file was specified `src/components/lavagna/ComponentDrawer.jsx` in the brief but actually lives at `src/components/simulator/panels/ComponentDrawer.jsx`. Maker-2 used the actual path; this is consistent with file ownership intent (the brief said the file in question).

### H2 — Hash route #lavagna-solo

**Problem**: Mandate 4 wants HomePage "Lavagna libera" card → /lavagna with NO UNLIM panel + NO ComponentDrawer + NO LessonReader. Today /lavagna shows everything.

**Solution**: LavagnaShell reads `window.location.hash === '#lavagna-solo'` on mount + listens to `hashchange`. When `lavagnaSoloMode` is true:
- `data-elab-mode="lavagna-solo"` set on shell wrapper (CSS targeting + E2E hook)
- GalileoAdapter NOT rendered
- PercorsoCapitoloView NOT rendered
- Passo Passo FloatingWindowCommon NOT rendered
- PercorsoPanel NOT rendered (guard added to N1)

**Trade-off**: WebDesigner-1 owns H1 (HomePage card click → set hash). Maker-2 only consumes the hash. If H1 not shipped this iter, hash route is dormant but harmless (no regression).

### L3 — beforeunload + pagehide sync flush

**Problem**: Andrea iter 19 PM bug "scritti spariscono su Esci" persists beyond iter 34 F1 (unmount flush) because page-close (browser X / hard refresh) does NOT trigger React unmount before navigation. Async supabase upsert is cancelled by browser on unload.

**Solution**:
1. `flushDebouncedSaveSync(experimentId, paths)` NEW export in `drawingSync.js` — clears debounce timer, fires fire-and-forget `savePaths` upsert.
2. `DrawingOverlay` adds `useEffect` mounting `beforeunload` + `pagehide` listeners (gated by `syncEnabled`) that call `flushDebouncedSaveSync` on page-close paths.

**Honest caveats**:
- Supabase JS client does NOT expose direct `sendBeacon` integration; we rely on the browser's `keepalive` flag handling (Chrome/Edge 64+, Firefox 65+, Safari 11.1+). For older browsers, fire-and-forget POST may be cancelled by the browser.
- Direct `navigator.sendBeacon()` to Supabase REST endpoint requires anon key in URL/header which is fragile vs the JS client; we kept the JS client path for now.
- This is **best effort** sync; if user is offline at the moment of close, paths are lost. NO retry queue (out of scope).

### N1 — Percorso 2-window overlay

**Problem**: Andrea: "Percorso deve corrispondere alla vecchia modalità libero ma ora ci sono 2 window sovrapposte". Today modalita==='percorso' opens UNLIM with 'percorso' tab but no separate Percorso panel; canvas may have stale experiment.

**Solution**:

4. PercorsoPanel uses existing FloatingWindow with localStorage persistence (z=10001 default > UNLIM 10000).
5. Dispatches `elab-lavagna-percorso-enter` CustomEvent for symmetry with G1 libero-enter.

**Trade-off**: PercorsoPanel was already imported (line 54) but never rendered — this is the FIRST integration of PercorsoPanel into the LavagnaShell render tree. WebDesigner-1 may need to polish the empty-state UI in N2.

### J1 — Percorso context payload (Sense 1.5 scaffold)

**Problem**: Mandate 6 wants UNLIM responses to adapt to docente + classe + livello + storia sessioni when in Percorso mode (Morfismo Sense 1.5).

**Solution**: `sendChat` accepts new `percorsoContext` option `{classKey, recentIntents, activeCapitolo, lastSessionDescriptions, kitLevel}`. When non-null, builds a `[CONTESTO PERCORSO — MORFISMO SENSE 1.5]` block and injects into the prompt sent to Edge Function `unlim-chat`.
