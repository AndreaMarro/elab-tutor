## CI/QUALITY GATE — 3 NUOVI PROBLEMI CRITICI

### [CRITICO] E2E Playwright non blocca mai la CI — silenziato con `|| echo`
**File**: `.github/workflows/e2e.yml`
```yaml
run: npx playwright test || echo "No E2E tests configured yet"
```
296 test E2E esistono (`e2e/*.spec.js`). La CI li esegue ma **ignora i fallimenti**. Un test E2E che crasha non fa fallire la PR. Il gate non protegge nulla. Questo è stato fatto evidentemente perché i test E2E non girano stabili — e invece di fixarli si è silenziato il gate.
**Conseguenza**: puoi mergiare su main con homepage rotta e CI verde.
**Fix**: Rimuovere `|| echo`. Fixare i test instabili. Effort: 2-4h.

### [ALTO] Baseline CI ratchet obsoleta di 2000 test

```json
{ "total": 1578, "date": "2026-04-09" }
```
Il benchmark dice 3611 test totali. La baseline è 1578 (aggiornata il 9/04). Il ratchet permette una regressione del 2% = circa 31 test. Ma se si scende da 3611 a 1579, il ratchet non scatta perché la baseline è sbagliata. Il numero usato per calcolare il minimo è **2x più basso della realtà**.
**Fix**: Aggiornare baseline a 3611. 5 minuti. Critico perché la protezione anti-regressione è fittizia.

### [ALTO] Bundle 12.6MB — 3.6x sopra il limite definito in baseline
